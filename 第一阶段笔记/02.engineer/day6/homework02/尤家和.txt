案例1:为虚拟机 server 配置以下虚拟Web主机
- 实现三个网站的部署
- 实现客户端访问server0.example.com网页内容为  复仇者联盟4
- 实现客户端访问www0.example.com网页内容为 琅琊榜30
- 实现客户端访问webapp0.example.com网页内容为 38里春风不如你

[root@server0 ~]# firewall-cmd --set-default-zone=trusted
success
[root@server0 ~]#  mkdir /var/www/ser0
[root@server0 ~]#  mkdir /var/www/www0
[root@server0 ~]#  mkdir /var/www/web0
[root@server0 ~]# vim /etc/httpd/conf.d/nsd01.conf
[root@server0 ~]# cat /etc/httpd/conf.d/nsd01.conf
<Virtualhost *:80>
ServerName server0.example.com
DocumentRoot /var/www/ser0
</Virtualhost>

<Virtualhost *:80>
ServerName www0.example.com
DocumentRoot /var/www/www0
</Virtualhost>

<Virtualhost *:80>
ServerName webapp0.example.com
DocumentRoot /var/www/web0
</Virtualhost>

[root@server0 ~]# echo '<h1>复仇者联盟4' > /var/www/ser0/index.html
[root@server0 ~]# echo '<h1>琅琊榜30' > /var/www/www0/index.html
[root@server0 ~]# echo '<h1>38里春风不如你' > /var/www/web0/index.html
[root@server0 ~]# systemctl restart httpd
[root@server0 ~]# vim /etc/httpd/conf.d/nsd01.conf
[root@server0 ~]# systemctl restart httpd


案例2:为虚拟机 server 配置Web访问控制
在 Web 网站 http://server0.example.com 的 DocumentRoot 目录下创建一个名为 private 的子目录，要求如下：
1）在server0.example.com的DocumentRoot目录下，private的子目录里书写网页文件index.html内容为  不许偷看丫
2）此页面只能在本机浏览，但是从其他系统不能访问这个目录的内容

虚拟机server0：
[root@server0 ~]# mkdir /var/www/ser0/private
[root@server0 ~]# vim /etc/httpd/conf.d/nsd02.conf
[root@server0 ~]# cat /etc/httpd/conf.d/nsd02.conf
<Directory /var/www/ser0/private>
Require ip 172.25.0.11
</Directory>
[root@server0 ~]# systemctl restart httpd
[root@server0 ~]# firefox server0.example.com/private
网页显示：
不许偷看丫


虚拟机desktop0：
[root@desktop0 ~]# firefox server0.example.com/private

网页显示：
Forbidden

You don't have permission to access /private on this server.


案例3:为虚拟机 server 使用自定Web根目录

调整 Web 站点 http://server0.example.com 的网页目录，要求如下：
1）新建目录 /webroot，作为此站点新的网页目录
2）确保站点 http://server0.example.com  仍然可访问

[root@server0 ~]# mkdir /webroot
[root@server0 ~]# vim /etc/httpd/conf.d/nsd01.conf 
[root@server0 ~]# cat /etc/httpd/conf.d/nsd01.conf
<Virtualhost *:80>
ServerName server0.example.com
DocumentRoot /webroot
</Virtualhost>

[root@server0 ~]# vim /etc/httpd/conf.d/nsd02.conf 
[root@server0 ~]# cat /etc/httpd/conf.d/nsd02.conf
<Directory /webroot>
Require all granted
</Directory>

[root@server0 ~]# chcon -R --reference=/var/www /webroot
[root@server0 ~]# ls -Zd /webroot
drwxr-xr-x. root root system_u:object_r:httpd_sys_content_t:s0 /webroot
[root@server0 ~]# systemctl restart httpd.service 

[root@server0 ~]# firefox server0.example.com

[root@server0 ~]# echo 'I am webroot' > /webroot/index.html
[root@server0 ~]# firefox server0.example.com



案例4：为虚拟机 server 部署动态WSGI站点
为站点 webapp0.example.com 配置提供动态Web内容，要求如下：
1）此虚拟主机侦听在端口8909
2）测试网页从以下地址下载，不要作任何更改http://classroom/pub/materials/webinfo.wsgi 
3）从浏览器访问 http://webapp0.example.com:8909 可接收到动态生成的 Web 页面

1)
[root@server0 ~]# vim /etc/httpd/conf.d/nsd01.conf
[root@server0 ~]# cat /etc/httpd/conf.d/nsd01.conf
Listen 8909
<Virtualhost *:8909>
ServerName webapp0.example.com
DocumentRoot /var/www/web0
</Virtualhost>

2)
[root@server0 web0]# wget http://classroom/pub/materials/webinfo.wsgi
[root@server0 web0]# yum -y install mod_wsgi.x86_64
[root@server0 web0]# vim /etc/httpd/conf.d/nsd01.conf
[root@server0 web0]# cat /etc/httpd/conf.d/nsd01.conf
Listen 8909
<Virtualhost *:8909>
ServerName webapp0.example.com
DocumentRoot /var/www/web0
WsgiScriptAlias / /var/www/web0/webinfo.wsgi
</Virtualhost>
[root@server0 web0]# semange port -a -t http_port_t -p tcp 8909
[root@server0 web0]# cat /etc/httpd/conf.d/nsd01.conf

[root@server0 web0]# firefox webapp0.example.com:8909
网页显示：
UNIX EPOCH time is now: 1531741011.6
